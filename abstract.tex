The security of the RSA cryptosystem relies on the random generation of large prime numbers. However, implementation flaws and low entropy issues lead to the generation of keys with shared prime factors, making them highly vulnerable to Greatest Common Divisor (GCD) attacks \cite{pelofske2023}. Auditing this vulnerability on a global scale using Certificate Transparency (CT) logs requires All-to-All Batch GCD algorithms. Previous research analyzed over 159 million public keys using the traditional Remainder Tree approach \cite{vage2022, pelofske2023}. While effective, this method incurred massive computational costs, requiring 88 execution hours and peaks of 180 GB of RAM \cite{vage2022}.
\\
\\
This Master's Thesis proposes optimizing this process by implementing the novel Binary Tree Batch GCD algorithm in C++ \cite{pelofske2023}. This method completely bypasses the computationally expensive remainder tree by computing GCD operations directly during the product tree construction \cite{pelofske2023}. The main objective is to replicate the large-scale analysis of CT logs to detect vulnerable certificates, empirically demonstrating that the new implementation drastically reduces memory consumption and accelerates execution time. The results of this work aim to show that it is possible to conduct cryptographic audits of modern web infrastructure in a much more efficient and accessible manner.
